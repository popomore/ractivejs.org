Components are a way of encapsulating complex functionality within a Ractive.js template. Conceptually, they're not dissimilar to Web Components, Ember.Component or Angular directives.

In this example, we've created a <rv-donutchart/> component. Each chart is a self-contained, isolated unit, but it can observe data from the surrounding context via its attributes. Look at the template and javascript code to see how this works in practice. (The data-binding works in both directions – we're not using it here, but a component can change data that it has 'access' to.)

Components can also fire events, which the parent ractive can listen to in the normal way. In this case we're listening for the select event – which each donut fires when the user moves over a segment – and responding by setting the selected property to the ID of the segment under the pointer.
