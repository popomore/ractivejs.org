This is an adaptation of the TodoMVC application. Ractive.js doesn't take a side in the MVC wars – it aims to be architecturally agnostic – so whereas other TodoMVC implementations have a strict separation between models, views and controllers, Ractive.js lets you structure your app how you see fit.

Often, your 'model' will be a plain old JavaScript object or array. Array mutator methods (push, pop, shift, unshift, splice, reverse and sort) are wrapped (by default – this behaviour can be prevented) so that the view gets updated with the least amount of work. See the wiki page on array modification for more info.

This example also demonstrates proxy events – which translate DOM events (such as mouse and keyboard interactions) into domain events (such as 'add item' or 'mark all as complete') – and transitions, which automate things like the slide down/up animations when you add/remove a todo.
